Technical Requirements

System Requirements

Everything your IT team needs to evaluate Reality Anchors.

Web Dashboard

Access the Reality Anchors dashboard from any modern browser — no installation required.

BrowserChrome 90+, Edge 90+, Firefox 90+, Safari 15+
ConnectionStable internet connection (WebSocket support required)
InstallationNone — SaaS delivery

Mobile Capture App

Beta

The Reality Anchors mobile app captures structured bend data from the shop floor. Currently available on Android.

OSAndroid 10 (API level 29) or later
CameraRear-facing camera, 12 MP or higher recommended
Depth sensorOptional — LiDAR or depth sensor enables advanced measurement features
ConnectivityWi-Fi or cellular data (for cloud upload)
StorageMinimum 2 GB free for local capture buffer

iOS: Not yet available. iOS is on our roadmap — contact us for timeline.

Network & Infrastructure

Reality Anchors is fully cloud-hosted. No on-premise hardware or VPN required.

Outbound accessHTTPS (port 443) to Reality Anchors cloud services
On-premise serverNot required — fully cloud-hosted
Data residencyUnited States (Google Cloud)

Security Architecture

Reality Anchors is built on an enterprise security architecture.

Role-based access control

Operators, managers, and admins see only what their role permits.

Encryption

Data encrypted in transit (TLS 1.2+) and at rest (AES-256).

Audit logging

Access and configuration changes are logged.

Authentication

Cloud-based identity management; SSO available on Production and Enterprise plans.
